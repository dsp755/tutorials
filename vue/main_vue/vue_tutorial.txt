Установка

1. npm i -g @vue/cli

2. vue create test   === создает папку с именем test

3. manually - выбираем фичи пробелом ==  можно выбрать vuex, он сохраняет данные в local storage

4. use history -y    === чтобы страницы не перезагружались, а работало как сингл-пейдж-апп

5. выбираем препроцессор - лучше сначала Less

6. выбираем линтер - стандартный. - Lint on save

7. выбираем где хранить конфиги - отдельными файлами или в package.json

8. можно сохранить эту настройку как темплейт для следующих проектов


Использование

1. Заходим в созданную папку в vs code

2. npm run serve === эта команда прописана в package.json === serve для разработки, а build для продакшена

3. serve показывает порт, на котором он запустился

4. открываем в браузере этот адрес


Структура

1. Точка входа приложения в папке src main.js. == обычный js файл с настроенным babel

2. В файле main.js крепятся компоненты == корневой компонент уже импортирован, это App.vue

3. В файле main.js по дефолту инициилизируется приложение === new Vue({ ... })

4. В папке components находятся элементы интерфейса === внутри тега template должен быть один тег (напр. div), а в нем уже любая вложенность.

5. папка scr > assets  в ней можно хранить картинки и т.п.

6. в тэге style любого компонента можно указать аттрибут lang="less" (если мы выбрали препроцессор less), тогда мы будем писать не чистый css, а less

7. при импортах во вью @ всегда указывает на папку src

8. если нам нужны компоненты из других папок, мы должны импортировать их внутри тэга script, затем -> 
в каждом компоненте должен быть export default - это и есть наш компонент. в нем прописывается в объекте имя компонента и другие его параметры - components - сюда мы подключаем детальки, которые мы можем использовать в тэге template, т.е. внутри разметки.
